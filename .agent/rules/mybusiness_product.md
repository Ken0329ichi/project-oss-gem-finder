### 3-3. セールスファネル設計（貢献・支援ファネル）

#### ステップ1: リード獲得
- **無料オファー**: サクサク動くOSSトレンド探索・面白タグ検索機能。
- **具体的な施策**: GitHub Pagesを通じ、クリーンで中立的なデータツールとしてギーク向けコミュニティにひっそりと露出。

#### ステップ2: 信頼構築
- **提供する価値**: サクサクした動作、ノイズのない中立なデータ表示。
- **具体的な内容**: 日常的なブックマークツールとしての利用を促進。

#### ステップ3: 教育
- **Belief Building**: 「データの中立性」と「ソース非公開・データCC BY 4.0による利活用自由度」を明示。自サイトは分析対象から排除する。

#### ステップ4: オファー（アクション）
- **セールス方法**: 自発的なコントリビューションの推奨。
- **具体的な施策**: ギークたちが「グレーゾーンの回収（README改善など）」という形で、他者のOSSをブラッシュアップするコントリビュート行動を起こすよう促す。
