# ビジネスルール (市場分析・ポジショニング)

> このファイルは、OSS Gem Finderプロジェクトの市場環境、競合状況、およびポジショニング戦略を定義したものです。

---

## 📊 1. 市場分析

### 1-1. 市場規模と成長性

**市場規模**: 
ニッチ（開発者エコシステムインテリジェンス単体では数億円〜数十億円規模）。ただし、OSSのサプライチェーンセキュリティ（企業の技術選定リスク判定）などの実用的なB2Bニーズに結びつくことで、実質的な市場は拡大する余地がある。

**成長トレンド**: 
急成長中（特にAI時代における技術選定の迅速化とセキュリティ観点での需要増）。

**市場の特徴**:
- 単なる「スター数の推移」だけでなく、「フォークに対するアクティブコントリビューター数の比率」「国別の開発コミュニティの強み」「AIコードアシスタント導入による開発速度の変化」など、より深層的なメトリクスを可視化するニーズが高まっている。
- 企業がOSSを採用する前の「健康診断」やセキュリティ・メンテナンス状況の確認ツールとしての需要。

### 1-2. 競合分析

#### 競合1: OSSInsight
- **強み**: GitHubの全イベント（50億行以上）を対象にしたリアルタイムかつ極めて多角的な分析。AIによる自然言語クエリ生成など高機能。
- **弱み**: バックエンドに大規模な分散型データベース（TiDB Cloud）を使用しており、維持コストやアーキテクチャが極めて重い。多機能すぎて難解。
- **価格帯**: 基本無料（TiDB Cloudのショーケースとして運営）。
- **ターゲット層**: 企業の技術選定担当者、投資家、テックトレンド調査員。

#### 競合2: GitHub Innovation Graph
- **強み**: GitHub公式が提供する、国別・経済圏別のコラボレーションやトレンドのマクロデータ。スパム等が排除された高い信頼性。
- **弱み**: 個別のリポジトリや開発者のミクロな分析は不可。データの更新が四半期（3ヶ月）ごとで、リアルタイム性に欠ける。
- **価格帯**: 完全無料。
- **ターゲット層**: 研究者、政策立案者、マクロテックトレンド調査者。

#### 競合3: Star History
- **強み**: スター数の推移のみに特化し、シンプルでサクサク動作する。xkcd風のグラフで手軽にリポジトリの成長を比較できる。
- **弱み**: 「スター数」という単一の指標しか見られないため、Issueやフォーク数、国別の貢献度、技術的空白領域などの深い分析はできない。
- **価格帯**: 完全無料（知名度向上・認知獲得ツールとして機能）。
- **ターゲット層**: 一般の開発者、自作OSSをアピールしたい管理者。

### 1-3. ポジショニングマップ

**横軸**: トーン（お堅い/単一指標 ↔ エンタメ・多角的な探索）
**縦軸**: システム動作（重い・高コスト ↔ 軽い・サクサク・低コスト）

```text
縦軸: システム動作（軽い・サクサク）
     ↑
     |  [Star History]
     |  (Starのみ・軽量)             ★ [あなた (Ken0329)]
     |                              (エンタメ多角・超軽量)
     |
     |  [Innovation Graph]
     |  (マクロ分析)                 [OSSInsight]
     |                              (多角分析・重いDB)
     +--------------------------------------------------→ 横軸: トーン（エンタメ・多角的）
```

**ポジショニングの解釈**:
既存のツールが「お堅い多角（重い）」か「お堅い単一（軽い）」に二分される中、面白タグやIssue活発度といった「エンタメ的で多角的な探索」と、「サーバーレスによる圧倒的な軽量動作」を両立するポジション。

**独自ポジション（極限の引き算と自律稼働）**:
GitHub PagesやActionsなどの無料インフラ枠を徹底活用し、維持コストを完全にゼロ（0円）に抑える「引き算の美学」を追求するポジション。無駄なデータベースや過剰なキャッシュ機構などの外付けギミック（複雑さ）を徹底的に排除（KISS/YAGNIの適用）し、広告やオピニオンを排して中立的かつクリーンな状態で、サクサク動くデータ可視化を提供します。
